Requirimientos de seguridad

Como se ha mencionado en artículos anteriores, es necesario mantener la seguridad del sistema para evitar posibles amenazas, ataques, caidas, fugas de datos, etc. Existen diversos métodos, siendo algunos principales y básicos, mientras que otros son opcionales y bastante más complejos.

Mantener actualizado el sistema

Debemos de actualizar cada cierto intervalo de tiempo el sistema para protegerlo de nuevas amenazas y corregir vulnerabilidades antiguas que pueden ser explotadas, además de que podremos obtener nuevas funciones en ciertos casos y mejoras de rendimiento y consumo.

Para ello nos dirigimos a la configuración del sistema y seleccionamos la opción de Actualización y seguridad. Una buena práctica es informarse de las nuevas actualizaciones y los problemas que solucionan, beneficios que otrogan, posibles errores, etc.

Realizar copias de seguridad

Dado que probablemente en algun momento perdamos nuestros datos o debamos de volver a restaurarlos, siempre es necesario realizar copias de seguridad a intervalos regulares ( diarias, semanales, cada 15 día o mensuales) de los datos más importantes, evitando así su pérdida y facilitando poder volver a restaurar el sistema en caso de fallos.

Es necesario tener instalado el Rol de copias de seguridad en el dominio. En caso de no tenerlo, deberemos de acceder a la opción de añadir roles y características y seguir los pasos necesarios, los cuales son bastante sencillos y carecen de complejidad.

Para realizar las copias de seguridad debemos de dirigirnos al Administrador del Servidor y en Herramientas, seleccionaremos la opción de Copias de seguridad de Windows Server.

Podemos programar las copias de seguridad para que se efectuen en un cierto momento, realizarlas manualmente, elegir si dicha copia será del servidor completo o parte de sus datos y como ya viene siendo de costumbre, la ubicación, pudiendo ser local o remota.

Crear reglas en el Firewall

El Firewall es una utilidad que viene implementada en los equipos Windows y que nos da la posibilidad de establecer reglas o parámetros para mejorar la seguridad de nuestros equipos, existen algunas diferencias entre el firewall de equipos comunes y el firewall de servidores.

El Firewall de Windows con seguridad avanzada nos permite definir valores y parámetros para el tráfico que es transmitido entre el equipo y la red. Podemos ver en el costado izquierdo que contamos con tres tipos de reglas que son:

  • Reglas de entrada: gestionan todo lo relacionado con el tráfico entrante a la máquina, por ejemplo, podemos bloquear que una máquina haga ping o nslookup a nuestro servidor o impedir conexiones remotas.
  • Reglas de salida: crea parámetros para las conexiones que son originadas desde el servidor hacia la red externa.
  • Reglas de seguridad de conexión: otorga mayor protección en el tráfico entre dos dispositivos.

Al crear reglas personalizadas podemos crear un sistema más afín a nuestras necesidades, pudiendo escepecificar el tipo de protovolo sobre el que se aplicará, el número del puerto local y remoto, configurar ICMP, direcciones IP locales y remotas sobre las que se aplicará, etc.
